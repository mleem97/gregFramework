## Stage 1: Documentation and metadata contract

- Define schema for mod manifests.
- Document contributor submission path.

## Stage 2: Validation pipeline

- Add CI validation for schema and dependency compatibility.
- Add mandatory hash and license checks.

## Stage 3: Security scanning

- Add virus scanning for built/attached artifacts.
- Record scan reports and approval state.

## Stage 4: Store frontend + API

- Docusaurus site consumes signed metadata index.
- Search/filter by game version/framework compatibility.

## Stage 5: Publisher trust and signing

- Optional maintainer signing keys.
- Reputation/trust tiers for publishers.
