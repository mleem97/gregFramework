# 🧭 UI Reference Notes

This document summarizes UI-related investigation guidance for `DataCenterExporter`.

---

## 🎯 Purpose

Use this file as a lightweight reference when mapping in-game UI paths for runtime logging and mod integration.

---

## 🔍 Recommended UI Discovery Workflow

1. Launch the game and open the target UI screen.
2. Use the in-game UI path hotkey (`F9`) to log the hierarchy under the mouse cursor.
3. Collect and compare multiple logs for the same panel/state.
4. Keep only stable paths that are reproducible across sessions.

---

## 🧩 Typical Data to Record

When documenting UI references, capture:

- Scene name
- Top-level canvas name
- Full object path
- Relevant component type(s)
- Notes about visibility/state requirements

---

## 📝 Suggested Entry Template

```text
Scene: <SceneName>
Canvas: <CanvasName>
Path: <Root/Panel/.../Target>
Components: <Image, Button, TMP_Text, ...>
State notes: <When this object appears or updates>
```

---

## ⚠️ Notes

- Keep this file human-readable; avoid storing massive autogenerated symbol dumps directly in markdown.
- Store large raw extraction outputs in separate artifacts/log files if needed.
- Prefer concise, reproducible UI paths over exhaustive low-level listings.
